# CHANGELOG

All notable changes to this project will be documented in this file.
The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## Unreleased

### Added
-   Support for CoAP (Constrained Application Protocol) as a transport for CMP messages.
-   Option to output test results in JSON format for easier integration with CI/CD pipelines.
-   Docker containerization for simplified setup and execution across different environments.
-   Enhanced error parsing for specific CMP failure information (e.g., `PKIStatus` values).

### Changed
-   Refactored `build_cmp_options` to dynamically include TLS client authentication parameters.

### Fixed
-   Issue with `openssl cmp` command sometimes hanging on specific server responses.

## 1.0.0 - 2026-02-24

### Added
-   Initial release of the `test_cmpv2_server.sh` script.
-   Core functionality for testing CMPv2 server operations:
    -   Initialization Request (IR)
    -   Certificate Request (CR)
    -   Key Update Request (KUR)
    -   Revocation Request (RR)
    -   General Message (GENM) for CA certificate retrieval
    -   PKCS#10 Certificate Request (P10CR)
    -   Root CA Certificate Update
-   Support for two primary CMP authentication methods:
    -   MAC-based authentication using a shared secret and reference.
    -   Signature-based authentication using a client certificate and private key.
-   Automatic generation of RSA or ECDSA key pairs for certificate requests.
-   Configurable server parameters: hostname, port, and URL path.
-   Optional TLS (HTTPS) support for secure communication, including server CA certificate trust.
-   Detailed logging to `cmpv2_test.log` within a dedicated output directory.
-   Verbose output option (`-v`) for real-time debugging information.
-   Pre-flight checks for OpenSSL version (ensuring 3.0+) and CMP module availability.
-   Connectivity tests (TCP/HTTP) to the CMP server.
-   Modular test execution, allowing users to select specific CMP operations to run.
-   Comprehensive command-line argument parsing and environment variable support for configuration.
-   Clear test summary indicating passed, failed, and skipped tests.
-   Creation of an organized output directory structure for generated certificates, keys, and CSRs.

### Changed
-   Default output directory changed to `./cmpv2_test_output`.
-   Improved error handling and logging messages for better user feedback.

### Fixed
-   N/A (Initial release)
